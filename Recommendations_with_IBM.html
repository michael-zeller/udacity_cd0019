def find_similar_users(user_id, user_item=user_item):
    '''
    INPUT:
    user_id - (int) a user_id
    user_item - (pandas dataframe) matrix of users by articles: 
                1's when a user has interacted with an article, 0 otherwise
    
    OUTPUT:
    similar_users - (list) an ordered list where the closest users (largest dot product users)
                    are listed first
    
    Description:
    Computes the similarity of every pair of users based on the dot product
    Returns an ordered
    
    '''
    # Compute similarity of each user to the provided user
    similarity = user_item.dot(user_item.loc[user_id])

    # Sort by similarity
    similarity = similarity.sort_values(ascending=False)

    # Create list of just the ids
    most_similar_users = similarity.index.tolist()

    # Remove the own user's id
    most_similar_users.remove(user_id)
    
    
       
    return most_similar_users # return a list of the users in order from most to least similar

def get_article_names(article_ids, df=df):

    OUTPUT:
    article_names - (list) a list of article names associated with the list of article ids

    OUTPUT:

    Description:
    Loops through the users based on closeness to the input user_id

    Does this until m recommendations are found

def get_top_sorted_users(user_id, df=df, user_item=user_item):
    '''
    INPUT:
    user_id - (int)
    df - (pandas dataframe) df as defined at the top of the notebook 
    user_item - (pandas dataframe) matrix of users by articles: 
            1's when a user has interacted with an article, 0 otherwise
    
            
    OUTPUT:
    neighbors_df - (pandas dataframe) a dataframe with:
                    neighbor_id - is a neighbor user_id
                    similarity - measure of the similarity of each user to the provided user_id
                    num_interactions - the number of articles viewed by the user - if a u
                    
    Other Details - sort the neighbors_df by the similarity and then by number of interactions where 
                    highest of each is higher in the dataframe
     
    '''
    
    # Compute similarity of each user to the provided user
    similarity = user_item.dot(user_item.loc[user_id])

    # Create a dataframe with the similarity and number of interactions
    neighbors_df = pd.DataFrame({'neighbor_id': similarity.index, 'similarity': similarity.values})
    # the number of interactions is computed by counting the number of occurences of the user_id in the df dataframe
    neighbors_df['num_interactions'] = df['user_id'].value_counts().loc[neighbors_df['neighbor_id']].values

    # Remove the own user's id
    neighbors_df = neighbors_df[neighbors_df['neighbor_id'] != user_id]

    # Sort by similarity and then by number of interactions
    neighbors_df = neighbors_df.sort_values(by=['similarity', 'num_interactions'], ascending=False)
    return neighbors_df # Return the dataframe specified in the doc_string


def user_user_recs_part2(user_id, m=10):
    '''
    INPUT:
    user_id - (int) a user id
    m - (int) the number of recommendations you want for the user
    
    OUTPUT:
    recs - (list) a list of recommendations for the user by article id
    rec_names - (list) a list of recommendations for the user by article title
    
    Description:
    Loops through the users based on closeness to the input user_id
    For each user - finds articles the user hasn't seen before and provides them as recs
    Does this until m recommendations are found
    
    Notes:
    * Choose the users that have the most total article interactions 
    before choosing those with fewer article interactions.

    * Choose articles with the articles with the most total interactions 
    before choosing those with fewer total interactions. 
   
    '''
    # Get the top sorted users
    neighbors_df = get_top_sorted_users(user_id, df, user_item)
    
    # Get the articles seen by the user
    seen_articles = get_user_articles(user_id, user_item)[0]
    
    recs = []
    for neighbor_id in neighbors_df['neighbor_id']:
        neighbor_articles = get_user_articles(neighbor_id, user_item)[0]
        new_recs = list(set(neighbor_articles) - set(seen_articles) - set(recs))
        recs.extend(new_recs)
        if len(recs) >= m:
            break
    
    recs = recs[:m]
    rec_names = get_article_names(recs, df)
    
    return recs, rec_names

Designing an Experiment to Evaluate the Recommendation Engine¶

To effectively evaluate the performance of the recommendation engine, we need to design a robust experiment. Here are the key aspects to consider:

1. Defining Evaluation Metrics¶

  • Precision and Recall: Measure the accuracy of the recommendations by calculating the proportion of recommended articles that are relevant (precision) and the proportion of relevant articles that are recommended (recall).
  • F1 Score: A harmonic mean of precision and recall, providing a single metric to evaluate the balance between them.
  • Mean Average Precision (MAP): Calculate the average precision for each user and then take the mean across all users.
  • Click-Through Rate (CTR): Measure the proportion of recommended articles that users click on.
  • User Engagement: Track metrics such as time spent on recommended articles, number of interactions, and user satisfaction surveys.

2. Setting Up Control Groups¶

  • Control Group: Users who receive recommendations from the existing system or a baseline model.
  • Treatment Group: Users who receive recommendations from the new recommendation engine.
  • Ensure that users are randomly assigned to either group to eliminate selection bias.

3. Implementing A/B Testing or Randomized Controlled Trials (RCT)¶

  • A/B Testing: Compare the performance of the new recommendation engine (treatment) against the existing system (control) by randomly assigning users to each group and measuring the defined metrics.
  • Randomized Controlled Trials (RCT): Similar to A/B testing but can include multiple treatment groups to test different variations of the recommendation engine.

4. Potential Challenges and Limitations¶

  • Cold Start Problem: New users or items with little interaction data may receive less accurate recommendations. Mitigate this by using hybrid models that combine collaborative filtering with content-based filtering.
  • Data Sparsity: Sparse interaction data can lead to poor model performance. Address this by incorporating additional data sources or using matrix factorization techniques.
  • User Privacy: Ensure that user data is anonymized and handled in compliance with privacy regulations.
  • Longitudinal Effects: Short-term experiments may not capture long-term user behavior changes. Consider running experiments for an extended period to observe sustained impacts.

5. Strategies to Mitigate Challenges¶

  • Hybrid Models: Combine collaborative filtering with content-based filtering to improve recommendations for new users and items.
  • Regularization: Apply regularization techniques to prevent overfitting and improve model generalization.
  • Cross-Validation: Use cross-validation to ensure that the model performs well on different subsets of the data.
  • User Feedback: Continuously collect and incorporate user feedback to refine the recommendation engine.

By carefully designing the experiment and addressing potential challenges, we can effectively evaluate the performance of the recommendation engine and make data-driven decisions to enhance its effectiveness.
